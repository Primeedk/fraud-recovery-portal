# Fraud Recovery & Investigation Program (FRIP) Portal

A high-fidelity, secure, and user-friendly multi-step portal designed for victims of fraud to securely submit details, digital evidence, and transaction records to recovery specialists. 

This application uses a supportive, stress-reducing design and official branding style guidelines to ensure trustworthiness and accessibility.

## 🚀 Features

- **6-Step Application Wizard**:
  1. **Victim Information**: Basic details and contact information.
  2. **Incident Details**: Type of fraud, dates, narrative description.
  3. **Financial Transactions**: Dynamic row additions for tracking transactions across multiple dates/amounts.
  4. **Fraudster Details**: Information on the suspect (phone numbers, emails, websites).
  5. **Evidence Vault**: Drag-and-drop file uploader with real-time simulated progress.
  6. **Consent & Signature**: HTML5 Canvas-based signature pad for consent verification.
- **Auto-Save Progress**: Caches progress locally using `localStorage` so victims can resume if they close the browser.
- **Responsive Layout**: Optimized for both desktop displays and mobile screens.
- **Premium Aesthetics**: Features smooth transitions, clean typography, secure badges, and a custom dark statistics panel showing impact metrics.

## 🛠️ Technology Stack

- **Frontend**: Vanilla HTML5, CSS3 (Custom Variables, Flexbox/Grid), and Modern ES6+ JavaScript.
- **Libraries**: Zero dependencies. Completely lightweight and fast-loading.

## 💻 Running the Project Locally

To run the project, simply open `index.html` in any web browser, or serve it locally:

```bash
# Using Python
python3 -m http.server 8000
```
Then visit `http://localhost:8000` in your browser.

## 🛡️ Admin Control Center

An administrative panel is included at `admin.html` to audit submitted reports. It features:
- **Real-time Analytics**: Counter metrics summarizing total cases, pending actions, and total loss amounts.
- **Advanced Search & Filtering**: Locate files by Case Reference ID or victim name, and filter cases by status or fraud category.
- **Detailed Folders View**: Inspect personal files, timelines, transaction tables, and digital biometrics traces (signature).
- **Pipeline Coordinator**: Update case statuses (e.g., *Under Investigation*, *Recovering Funds*) in real-time.

To view the dashboard, open `admin.html` in your browser, or visit `http://localhost:8000/admin.html` while serving the project.

## 🗄️ Database Integration (Supabase Setup)

By default, submissions are stored inside the browser's `localStorage` so the system works instantly. To connect it to a cloud database:

1. Create a project at [Supabase.com](https://supabase.com).
2. Open the **SQL Editor** in your Supabase dashboard and run this query to create the table structure:

```sql
CREATE TABLE frip_submissions (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  case_ref text UNIQUE NOT NULL,
  submitted_at timestamp with time zone NOT NULL,
  status text NOT NULL DEFAULT 'Pending Review',
  
  -- Victim details
  victim_name text NOT NULL,
  victim_dob date NOT NULL,
  victim_id_num text NOT NULL,
  victim_phone text NOT NULL,
  victim_email text NOT NULL,
  victim_address text NOT NULL,
  victim_occupation text NOT NULL,
  behalf_toggle boolean DEFAULT false,
  rep_relationship text,
  rep_auth text,

  -- Incident Details
  fraud_type text NOT NULL,
  fraud_contact_method text NOT NULL,
  fraud_start_date date NOT NULL,
  fraud_end_date date,
  fraud_ongoing boolean DEFAULT false,
  fraud_narrative text NOT NULL,

  -- Financial Details
  loss_amount numeric NOT NULL,
  loss_currency text NOT NULL DEFAULT 'USD',
  victim_bank_name text NOT NULL,
  victim_bank_acc_name text NOT NULL,
  victim_bank_acc_num text NOT NULL,
  victim_bank_ref text,

  -- Perpetrator Details
  fraudster_name text NOT NULL,
  fraudster_company text,
  fraudster_phone text,
  fraudster_email text,
  fraudster_website text,
  fraudster_social text,
  fraudster_comm_platform text NOT NULL,
  fraudster_bank text,
  fraudster_acc_num text,
  fraudster_swift text,
  fraudster_wallet text,

  -- Evidence Details
  contacted_bank text NOT NULL,
  bank_ref_num text,
  contacted_police text NOT NULL,
  police_ref_num text,

  -- Consent & Signatures
  consent_auth boolean DEFAULT false,
  consent_gdpr boolean DEFAULT false,
  consent_accuracy boolean DEFAULT false,
  consent_date date NOT NULL,
  signature_data text NOT NULL, -- Holds Base64 DataURL
  transactions jsonb NOT NULL DEFAULT '[]', -- Holds list of transactions
  uploaded_files jsonb NOT NULL DEFAULT '[]' -- Holds file meta lists
);

-- Enable Row Level Security (RLS)
ALTER TABLE frip_submissions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public insert" ON frip_submissions FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow public read/write admin" ON frip_submissions TO anon USING (true) WITH CHECK (true);
```

3. Go to **Project Settings > API** in Supabase and copy your **Project URL** and **anon public API key**.
4. Paste these values into `env.js` in your project folder.
5. Reload `admin.html` and `index.html` to sync.
